📋 Panduan: Cara Buat GitHub Token (PAT)
1
Login ke GitHub
Buka github.com dan pastikan sudah login ke akun kamu.
2
Buka Settings → Developer Settings
Klik foto profil kamu (kanan atas) → Settings → scroll ke bawah → Developer settings.
3
Buat Token Baru
Pilih Personal access tokens → Tokens (classic) → klik Generate new token (classic).
4
Isi & Pilih Permission
Beri nama token (contoh: cdn-yudaverse), atur expiration sesuai kebutuhan.
Centang permission: repo (full control of private repositories) — ini mencakup semua yang dibutuhkan.
5
Salin Token
Klik Generate token, lalu salin token-nya sekarang — token hanya ditampilkan sekali! Paste di kolom Token di bawah.
6
Buat Repository untuk CDN
Buat repo baru di GitHub (bisa public atau private). Pastikan sudah ada minimal 1 commit (init) agar branch main tersedia. Nama repo bebas, contoh: cdn-assets.
🐙 GitHub Settings
Username GitHub kamu (bukan email)
Nama repo yang akan jadi tempat penyimpanan file
Default: main — isi sesuai branch repo kamu
Folder di dalam repo. Kosongkan jika ingin file disimpan di root repo.
Token hanya disimpan di localStorage browser kamu, tidak dikirim ke server manapun selain GitHub.
Isi username, repo, branch, dan folder untuk melihat preview URL
ℹ️ Informasi
Semua konfigurasi disimpan di localStorage browser kamu.
Data tidak dikirim ke server manapun — hanya ke GitHub API saat upload.

⚠ Batas ukuran file GitHub: Maksimal 100MB per file. File di atas 50MB disarankan menggunakan Git LFS.
⚠ Repo publik: File bisa diakses siapa saja via raw URL. Gunakan repo private jika ingin akses terbatas (tapi raw URL tetap butuh token untuk diakses).
